**Prompt for AI UI Generator:**

**Project Concept:**
Generate a high-fidelity landing page design for a developer-focused technical product (e.g., a database, API, or authentication service). The aesthetic must be "Modern Technical Minimalist" or "Dark Engineering," resembling a futuristic Integrated Development Environment (IDE) or a heads-up display (HUD).

**Theme & Atmosphere:**

- **Overall Vibe:** Precise, geometric, technical, low-light, and highly structured. It should feel built _by_ engineers, _for_ engineers.
- **Color Palette (Structure):** Strictly dark mode. Use a deep charcoal or near-black background (#0a0a0a or similar). Containers should be slightly lighter matte slate grays. Use sharp, high-contrast white for primary text. Use a single, electric accent color (like cyan, neon purple, or electric blue) _sparingly_ for glowing borders, active states, and key visual highlights.

**Layout & Architecture:**

- **Strict Grid:** The layout must be governed by a visible, rigid grid.
- **Bento Box Structure:** Content must be compartmentalized into distinct, rectangular cards or modules with sharp, non-rounded corners.
- **Separation:** Separate sections with ultra-thin, low-contrast border lines (1px width) rather than shadows or depth effects, creating a blueprint-like feel.
- **Whitespace:** Despite the dark theme, maximize negative space between grid elements to ensure high readability and scannability.

**Typography & Text Styling:**

- **Font Pairing:** Use a clean, neutral neo-grotesque sans-serif (e.g., Inter, Satoshi) for headlines and body copy. Pair this with a distinct monospaced font (e.g., JetBrains Mono, Fira Code) for technical details, tags, and labels.
- **Hierarchy:** Use massive, bold H1 headlines that contrast dramatically with tiny, uppercase monospaced overlines or metadata tags.
- **Tech Formatting:** Frequently enclose sub-labels or keywords in brackets, such as `[API ROUTE]` or `{JSON CONFIG}`, to enhance the coding aesthetic.

**Key Visual Components:**

- **Hero Section:** A split-screen design. Left side: large headline and value proposition. Right side: An isometric, abstract wireframe diagram illustrating data flow, architecture nodes, or technical processes—not realistic illustrations.
- **Code as Content:** Treat code snippets as primary visual elements. Display them in dark terminal-style windows with distinct syntax highlighting, looking like active organizations.
- **UI Details:** Add subtle background textures like faint dot-matrix patterns or micro-grid lines in empty spaces. Use thin "laser" border glows on active input fields or high-priority containers.
- **Buttons:** Primary call-to-action (CTA) buttons should be solid geometric blocks. Secondary buttons should be "ghost" styles (transparent background with a thin border) or simple text links with a monospaced arrow `->`.
